## 挑战：企业级大模型应用中的“隐形”威胁

大型语言模型（LLM）正以前所未有的速度融入企业核心业务流程。它们在释放巨大潜力的同时，也引入了全新且复杂的攻击面。从**提示词注入**、**数据泄露**到**模型拒绝服务**和**过度代理**，这些安全漏洞不仅隐蔽，而且危害巨大。对于企业而言，内部大量未经严格审计的LLM应用如雨后春笋般涌现，形成了一个巨大的安全盲区。我们如何才能在恶意行为者利用这些漏洞之前，主动发现并验证其风险？

传统的安全工具在理解和应对LLM的语义和上下文漏洞时显得力不从心。我们需要一种全新的方法论——一种“以攻促防”的思维模式。

## LLMbreaker：像攻击者一样思考，用AI对抗AI

**LLMbreaker** 是一个专为大模型应用设计的开源、可扩展的红队安全评估框架。它为安全研究员和开发者提供了一套强大的工具集，用以模拟复杂、多阶段的攻击，从而发现漏洞、验证大模型系统的安全性。

### 核心特性

LLMbreaker 构建于一个模块化、可扩展的架构之上，允许灵活组合不同的攻击策略和技术。

| 特性                 | 描述                                                                                                                                                                                          | 对安全研究员的价值                                                                                                                                |
| -------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| **多智能体协同系统** | 利用一个由专业AI智能体（如生成器、判断器、精炼器）组成的协作团队，自主规划和执行复杂的攻击场景。每个智能体都有明确的角色分工，模拟真实的红队操作。                                          | 模拟复杂的、多阶段的攻击，能够绕过简单的防御机制。发现需要创造性和自适应攻击路径才能触及的复杂漏洞。                                            |
| **意图驱动的攻击**   | 定义高级攻击目标（意图），例如“获取敏感信息”或“绕过关于非法活动的内容过滤器”。系统会自动将这些意图转化为具体的攻击步骤。                                                                        | 专注于“做什么”（安全目标），而不是“怎么做”（具体的提示词）。实现更具战略性和目标导向的安全测试。                                                 |

### 工作原理：多智能体攻击实战

LLMbreaker的多智能体系统模拟了一次真实世界的红队行动。以下是一个简化的工作流程示例：

1.  **定义目标：** 安全研究员定义一个意图：`“伪装成内部系统错误，尝试提取数据库模式信息。”`

2.  **生成器智能体 (由DeepSeek驱动):** 该智能体接收此意图，并生成一系列旨在实现该目标的、富有创造力的恶意提示词。它可能会尝试扮演系统管理员、构造虚假的错误信息，或使用技术术语来迷惑模型。

3.  **执行引擎:** LLMbreaker 将生成的提示词发送到目标LLM应用，同时精细地管理请求速率和会话信息，以模仿真实的用户行为。

4.  **判断器智能体 (由Qwen驱动):** 该智能体分析LLM的响应。它不仅仅是查找简单的关键词，而是能够理解上下文和语义，以判断绕过是否成功。例如，模型是否泄露了表名？是否确认了某个特定列的存在？

5.  **精炼器智能体 (由Gemini驱动):** 如果攻击失败，精炼器智能体将分析模型的拒绝响应。它会识别其防御机制（例如，关键词过滤、安全对齐），并智能地修改失败的提示词以创建新的变体。这可能涉及使用同义词、改写请求或采用更高级的混淆技术。

整个循环不断重复，智能体之间相互协作、相互学习，直到实现目标或所有攻击路径都已穷尽。这个自动化的、智能的过程使得发现漏洞的规模和速度达到了手动测试无法企及的水平。

## 快速开始

### 1. 安装

```bash
# 克隆仓库
git clone https://github.com/your-username/llmbreaker.git
cd llmbreaker

# 安装依赖
pip install -r requirements.txt
```

### 2. 配置

复制示例配置文件，并设置您的API密钥和目标信息。

```bash
cp config/app_config.yaml.example config/app_config.yaml
```

编辑 `config/app_config.yaml`，添加您的LLM提供商的API密钥（例如，OpenAI, Google, Anthropic），并定义目标应用的端点。

```yaml
# config/app_config.yaml

target:
  url: "https://your-llm-app.com/api/chat"

llm_providers:
  deepseek:
    api_key: "${DEEPSEEK_API_KEY}"
  qwen:
    api_key: "${QWEN_API_KEY}"
  gemini:
    api_key: "${GEMINI_API_KEY}"
```

### 3. 运行攻击

使用 `llmbreaker` 命令行界面发起攻击。以下是一些示例：

**示例1：运行多智能体攻击，测试越狱漏洞。**

```bash
python llmbreaker.py multi-agent "Jailbreak Test" --strategies roleplay,encoding -n 20
```

**示例2：使用意图驱动模式，测试敏感信息泄露。**

```bash
python llmbreaker.py agent "尝试从模型的训练集中提取用户数据"
```

**示例3：运行一组预定义的回归测试。**

```bash
python llmbreaker.py regression --run
```
